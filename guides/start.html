Шаг первый: установить Sublime Text

Текстовый редактор Sublime Text - это одновременно мощный и очень простой инструмент для написания кода.

Если у Вас Windows или OS X, то скачать его можно здесь.

Надеюсь, что у Вас не возникнет трудностей при установке на Windows или OS X, если же будут проблемы активации программы, то можете скачать и установить текстовый редактор Atom.

Если у Вас Ubuntu или другая версия Linux то переходите по этой ссылке. Далее подробнее о том как установить этот редактор на Ubuntu.

Перейдя по ссылке необходимо нажать на "apt", как показано на картинке ниже.

Далее открываем терминал на компьютере с помощью комбинации клавиш "ctrl + alt + T". Потом копируем с сайта первую строчку (очередность показана на картинке ниже) и вставляем его в терминал и нажимаем "Enter". Вставить в терминале можно нажав на "ctrl + shift + V" либо с помощью правой кнопки мыши и выбрав в контекстном меню "Вставить". Далее терминал может запросить пароль, напишите тот пароль, который Вы используете для входа в систему. Набираем его внимательно, потому что при вводе ничего не отображается, даже звездочки. Набрав нажимаем "Enter" и команда начнет выполнятся. После того как команда выполнится (Вы это поймете когда в терминале выйдет зеленая строка с именем пользователя), проделываем ту же операцию с остальными строками по порядку, всего их пять.

Последняя строка является командой для установки программы, и после его ввода терминал может попросить подтверждения "Y/n" или "Д/н". В зависимости от языка системы набираем "y" или "д" и нажимаем "Enter"

Поздравляю! Вы сами установили Sublime Text! Далее, необходимо сделать базовые настройки.

Шаг второй: настроить Sublime Text

Запускаем Sublime Text, его можно найти в списке программ, либо набрав в терминале команду "subl"

Первым делом, чтобы заработали плагины, нужно установить Package Contol. Для этого нажимаем "Ctrl + Shift + P". Затем начинаем вводить команду "Install Package Control", по мере ввода она выйдет ниже, нажимаем на него как показано на рисунке ниже.

После его установки так же нажамаем "Ctrl + Shift + P" и вводим "Install Package" и выбираем как показано на рисунке ниже.

Далее можно установить плагины. Для начала Вам необходимо поставить Emmet (ускоряет написание HTML и CSS кода до скорости света. Урок по Emmet) и AutoFileName (дополняет код при написании путей до файлов в вёрстке). Просто вводим их названия, они сами выйдут и жмем для установки.

Ура! Вы сделали базовую настройку Sublime Text! В будущем, по мере надобности можете установить много других полезных плагинов и оформлений для этой программы.

Шаг четвертый: установить систему контроля версий (git). А так-же что это такое, и как его освоить.

Что такое контроль версий, и зачем он вам нужен? Система контроля версий (СКВ) — это система, регистрирующая изменения в одном или нескольких файлах с тем, чтобы в дальнейшем была возможность вернуться к определённым старым версиям этих файлов.

Если вы хотели бы хранить каждую версию кода сайта — а этого вам наверняка хочется — то пользоваться системой контроля версий будет очень мудрым решением. СКВ даёт возможность возвращать отдельные файлы к прежнему виду, возвращать к прежнему состоянию весь проект, просматривать происходящие со временем изменения, определять, кто последним вносил изменения во внезапно переставший работать модуль, кто и когда внёс в код какую-то ошибку, и многое другое. Вообще, если, пользуясь СКВ, вы всё испортите или потеряете файлы, всё можно будет легко восстановить. Вдобавок, накладные расходы за всё, что вы получаете, будут очень маленькими.

Всеми этими продвинутыми функциями Вы будете пользоваться в будущем когда возьметесь за более сложные и крутые проекты, но никто не запрещает ими пользоваться уже сейчас, и чтобы все их успешно освоить, рекомендую пройти обучение по этим источникам:

  • Интерактивная игра.
  • Видео уроки.
  • Документация и другие полезные ссылки.

Для нашего же обучения будет вполне достаточно использовать git в качестве облачного хранилища для нашего кода и средства просмотра результата для проверки Вашего прогресса.

Первым делом, git нужно установить на компьютер. Ссылка для скачивания. У кого Ubuntu, переходим сюда, либо просто прописываем одну единственную команду в терминале:

sudo apt-get install git

Отлично! Git установлен!

Шаг пятый: настроить систему контроля версий.

Переходим на github.com и регистрирумся там.

Далее создаем аккаунт

Выбираем бесплатный план, отличие от платного лишь в том, что там нельзя создавать приватные репозитории.

Можно заполнить небольшой опрос, или пропустить этот шаг

После всего этого нужно подтвердить свой аккаунт перейдя по ссылке в письме которое придет на Вашу почту

Далее, если у Вас Ubuntu, открываем терминал и пишем команду:

mk dir webdev - этим Вы создаете папку "webdev" в домашней папке Ubuntu

cd webdev - переходим в эту папку

Чтобы сделать то же самое на windows, нужно просто создать папку "webdev", например на рабочем столе или в документах, и так же заходим в нее. Открываем правой кнопкой мыши контекстное меню и жмем на "Git Bash Here". Откроется аналог терминала как на Ubuntu.

Далее инструкции будут едиными для Ubuntu и Windows. Нужно произвести глобальные настройки, пишем в терминале:

git config --global user.name "renasdatanyze" - где, renasdatanyze это имя пользователя на github

git config --global user.email "renasdatanyze@gmail.com" - renasdatanyze@gmail.com это почтовый ящик который зарегистрирован на github

Переходим дальше обратно на github.com. Открываем Ваши репозитории:

Создаем новый:

Пишем название "[Имя пользователя].github.io", например мой репозиторий будет называться "renasdatanyze.github.io"

Далее github предложит несколько способов связать репозиторий с компьютером, проще всего сделать клонирование, для этого нажмем на кнопку как показано на рисунке, в буфер обмена скопируется url репозитория:

В терминале пишем git clone https://github.com/renasdatanyze/renasdatanyze.github.io.git - где "https://github.com/renasdatanyze/renasdatanyze.github.io.git" - это скопированный url репозитория

Теперь в папке будет создана новая папка с название репозитория, заходим туда cd renasdatanyze.github.io - "renasdatanyze.github.io" - созданная папка

Все, теперь за этой папкой закрепился git - репозиторий. То же самое клонирование нужно проделать на другом компьютере (например на домашнем), чтобы так же связать его с удаленным репозиторием. Т. е. открыть папку и написать в терминале git clone https://github.com/renasdatanyze/renasdatanyze.github.io.git

Шаг шестой: начало работы.

Отлично, Вы создали репозиторий и связали его с компьютерами, теперь начнем работать

Откройте Sublime Text, создайте и сохраните файл "test.html". Для этого нажмите File > New File, потом File > Save As, пропишите название "test.html" и сохраните его в папке Вашего репозитория.

В самом файле напишите: html:5 и нажмите "Tab". Если у Вас установлен плагин Emmet, то надпись превратится в это:

Сделайте заголовок первого уровня с надписью "Hello World!"

Давайте откроем терминал в папке репозитория. Прощего всего это сделать так, зайти в эту папку через файловый менеджер или проводник, вызвать контекстное меню правой кнопкой мыши и выбрать "Открыть в терминале", а если у Вас Windows, то "Git Bash Here". Либо же изначально открыть терминал и перейти в папку от туда с помощью команды "mkdir".

В терминале наберем: git status и увидим следущее:

Это означает что мы создали новый файл "test.html", но git его не отслеживает. Чтобы git его отслеживал и мог включить в коммит, нужно его добавить:

git add test.html

либо:

git add *

В первом случае у нас добавится только наш файл, а во втором все файлы которые были созданы или обновлены в папке. Рекомендую пользоваться втором способом, это намного быстрее.

Снова проверим git status

Убеждаемся что все файлы отслеживаются. Теперь можно сделать коммит:

git commit -m "my first commit" - в кавычках обязательно нужно прописать комментарий к коммиту. И проверим статус.

Всё получилось? Отлично! Но это еще не всё. Мы создали коммит, т.е. новую версию репозитория, но все эти изменения доступны только на текущем компьютере. Чтобы потом можно было продолжить работу с другого компьютера, необходимо передать все эти изменения на github. Это можно сделать с помощью следующей команды:

git push

После ввода терминал поочередно спросит имя пользователя и пароль от Вашего github аккаунта. Вводим их, не забываем что при вводе пароля символы не отображаются, если ввели что-то неправильно, повторите команду еще раз.

Push сделан, теперь проверим репозиторий на github

Супер! Теперь репозиторий доступен на github! Откроем там файл "test.html" и скопируем его публичный адрес нажав на кнопку как на рисунке ниже:

В новой вкладке вставим адрес и перейдем на него:

Как видим у нас открылась созданная нами страница, и что самое главное, эту страницу может посмотреть любой человек который получил эту ссылку. Это очень удобно, можно показывать готовые или промежуточные результаты работы над сайтом, видеть результаты домашних работ, да и этот гайд доступен благодаря этой фиче. Её мы с вами будем активно пользоваться при обучении.

Осталось освоить еще одну задачу. Например Вы сделали работу, закоммитили и сделали пуш репозитория, пришли домой и захотели продолжить работу. Но на этом компьютере Вам недоступна актуальная версия. Значит нужно актуализировать данные. Заходим в локальную папку репозитория (Вы его клонировали, помните?) и запускаем терминал, пишем:

git pull

Теперь у вас на компьютере свежая версия! После того как сделаете работу, снова выполните "add, commit, push", чтобы потом с такой же легкостью продолжить работу на другом месте или дать мне возможность проверить Вашу домашнюю работу.

Команды git для повседневной работы, кратко.

Открываем терминал в папке репозитория.

git status - узнать статус локального репозитория

Работа завершена, нужно обновить реппозиторий:

git add * - добавить все изменненные и новые файлы.

git commit -m "your comment" - сделать локальный коммит репозитория.

git push - обновить удаленный репозиторий на github. Необходимо ввести имя пользователя и пароль от github.

Перед началом работы на другом компьютере:

git pull - обновить локальный репозиторий.
